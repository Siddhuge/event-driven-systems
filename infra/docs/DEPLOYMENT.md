# Terraform Deployment Guide\n\n## Quick Start\n\n### 1. Initialize Terraform\n\n```bash\ncd envs/dev  # or envs/prod\nterraform init\n```\n\n### 2. Create Variables File\n\n```bash\ncp terraform.tfvars.example terraform.tfvars\n# Edit terraform.tfvars with your values\n```\n\n### 3. Plan Deployment\n\n```bash\nterraform plan -out=tfplan\n```\n\nReview the plan carefully, especially:\n- New resources being created\n- Resource modifications\n- Deletions (should be rare)\n\n### 4. Apply Configuration\n\n```bash\nterraform apply tfplan\n```\n\n## Azure Setup\n\n### Prerequisites\n\n```bash\n# Install Azure CLI\ncurl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash\n\n# Install Terraform\nwget https://releases.hashicorp.com/terraform/1.5.0/terraform_1.5.0_linux_amd64.zip\nunzip terraform_1.5.0_linux_amd64.zip\nsudo mv terraform /usr/local/bin/\n\n# Verify installations\naz --version\nterraform --version\n```\n\n### Authentication\n\n#### Option 1: Authenticate with Azure CLI (Interactive)\n\n```bash\naz login\naz account set --subscription \"<SUBSCRIPTION_ID>\"\n```\n\n#### Option 2: Service Principal (CI/CD)\n\n```bash\n# Create service principal\naz ad sp create-for-rbac --name \"terraform-sp\" \\\n  --role \"Contributor\" \\\n  --scopes /subscriptions/<SUBSCRIPTION_ID>\n\n# Set environment variables\nexport ARM_CLIENT_ID=\"<appId>\"\nexport ARM_CLIENT_SECRET=\"<password>\"\nexport ARM_SUBSCRIPTION_ID=\"<SUBSCRIPTION_ID>\"\nexport ARM_TENANT_ID=\"<tenant>\"\n```\n\n## Environment Variables\n\n### Terraform-Specific\n\n```bash\n# Enable debug logging\nexport TF_LOG=DEBUG\nexport TF_LOG_PATH=/tmp/tf.log\n\n# Disable state lock (local dev only!)\nexport TF_SKIP_REMOTE_CONFIG_VALIDATION=true\n\n# Automatically approve apply (CI/CD only)\nexport TF_INPUT=false\n```\n\n### Azure Provider\n\n```bash\n# Required for scripted authentication\nexport ARM_SUBSCRIPTION_ID=\"<subscription-id>\"\nexport ARM_TENANT_ID=\"<tenant-id>\"\nexport ARM_CLIENT_ID=\"<client-id>\"\nexport ARM_CLIENT_SECRET=\"<secret>\"\n```\n\n## Workflow\n\n### Development Workflow\n\n```bash\n# 1. Check current state\nterraform state list\n\n# 2. Make changes to .tf files\nvim main.tf\n\n# 3. Format code\nterraform fmt -recursive ..\n\n# 4. Validate syntax\nterraform validate\n\n# 5. Plan changes\nterraform plan -out=tfplan\ncat tfplan  # Review\n\n# 6. Apply if satisfied\nterraform apply tfplan\n\n# 7. Verify resources created\naz resource list --resource-group <rg-name>\n```\n\n### Production Workflow\n\n```bash\n# Additional caution steps for prod\n\n# 1. Create backup of current state\nterraform state pull > prod.tfstate.backup\naz storage blob upload --account-name <backup-account> \\\n  --container-name tfstate --name prod.$(date +%Y%m%d).tfstate \\\n  --file prod.tfstate.backup\n\n# 2. Plan and export to file\nterraform plan -out=prod-$(date +%Y%m%d-%H%M%S).tfplan\n\n# 3. Code review (share plan output with team)\nterraform show <planfile>\n\n# 4. Only apply after approval\nterraform apply <planfile>\n\n# 5. Post-deployment verification\naz resource list --resource-group event-driven-prod-rg\n```\n\n## Common Tasks\n\n### List All Resources\n\n```bash\nterraform state list\nterraform state list 'module.aks.*'\n```\n\n### Show Resource Details\n\n```bash\nterraform state show module.aks.azurerm_kubernetes_cluster.aks\n```\n\n### Destroy Specific Resource\n\n```bash\n# Plan deletion\nterraform plan -destroy -target module.acr\n\n# Destroy if needed\nterraform destroy -target module.acr\n```\n\n### Import Existing Resource\n\n```bash\n# If resource exists in Azure but not in Terraform\naz resource show --ids /subscriptions/.../resourceGroups/mygroup\n\nterraform import azurerm_resource_group.rg /subscriptions/.../resourceGroups/mygroup\n```\n\n### Refresh State\n\n```bash\n# Sync state with actual Azure resources\nterraform refresh\nterraform plan  # Show any drift\n```\n\n### View Output Values\n\n```bash\nterraform output -json\nterraform output acr_login_server\n```\n\n## Troubleshooting\n\n### Enable Debug Logging\n\n```bash\nexport TF_LOG=DEBUG\nterraform plan 2>&1 | tee terraform.log\nunset TF_LOG\n```\n\n### State File Issues\n\n```bash\n# Check state validity\nterraform validate\n\n# View state structure\nterraform state list\nterraform state show <resource>\n\n# Force refresh\nterraform refresh\nterraform init -upgrade\n```\n\n### Common Errors\n\n#### \"Error: authenticating with Azure: \"\n\nSolution: Ensure Azure CLI is authenticated\n```bash\naz account show\naz login\n```\n\n#### \"Error: Unsupported argument\"\n\nSolution: Module variable names must match exactly\n```bash\nterraform validate  # Shows exact error\n```\n\n#### \"Error: Resource not found\"\n\nSolution: Resource deleted externally\n```bash\nterraform state rm <resource>\nterraform plan # Will recreate it\n```\n\n#### \"Error: Could not acquire state lock\"\n\nSolution: Another Terraform process running\n```bash\n# Wait for other process to finish, or\nterraform force-unlock <lock-id>  # Emergency only\n```\n\n### Debugging Kubernetes Integration\n\n```bash\n# Get kubeconfig after AKS creation\naz aks get-credentials --resource-group <rg> --name <cluster>\n\n# Test connectivity\nkubectl cluster-info\nkubectl get nodes\n\n# View AKS logs\naz aks command invoke --resource-group <rg> --name <cluster> \\\n  --command \"kubectl get nodes\"\n```\n\n## Backing Up State\n\n```bash\n# Local backup\nterraform state pull > terraform.tfstate.backup\n\n# Azure Storage backup (recommended)\naz storage account create --name tfstatebackup --resource-group mygroup\naz storage container create --account-name tfstatebackup --name backup\naz storage blob upload --account-name tfstatebackup --container-name backup \\\n  --name $(date +%Y%m%d).tfstate --file terraform.tfstate\n```\n\n## Cleaning Up\n\n### Destroy All Resources\n\n```bash\n# Plan destruction\nterraform plan -destroy\n\n# Destroy if confirmed\nterraform destroy\n\n# Verify in Azure\naz resource list --resource-group <rg-name>\n```\n\n### Retain Certain Resources\n\n```bash\n# Prevent destruction using lifecycle rule\nlifecycle {\n  prevent_destroy = true\n}\n```\n\n### Emergency Cleanup\n\n```bash\n# If state is corrupted, manually delete via Azure\naz group delete --resource-group <rg-name> --yes --no-wait\n```\n\n## CI/CD Integration\n\n### GitHub Actions Example\n\n```yaml\nname: Terraform\non: [push, pull_request]\n\njobs:\n  plan:\n    runs-on: ubuntu-latest\n    steps:\n      - uses: actions/checkout@v3\n      - uses: hashicorp/setup-terraform@v2\n        with:\n          terraform_version: 1.5.0\n      \n      - name: Terraform Plan\n        run: |\n          cd envs/dev\n          terraform init\n          terraform plan -out=tfplan\n        env:\n          ARM_CLIENT_ID: ${{ secrets.AZURE_CLIENT_ID }}\n          ARM_CLIENT_SECRET: ${{ secrets.AZURE_CLIENT_SECRET }}\n          ARM_SUBSCRIPTION_ID: ${{ secrets.AZURE_SUBSCRIPTION_ID }}\n          ARM_TENANT_ID: ${{ secrets.AZURE_TENANT_ID }}\n```\n\n## Performance Tuning\n\n```bash\n# Parallel resource provisioning (default: 10)\nterraform apply -parallelism=20\n\n# Skip certain checks for speed (dev only)\nterraform plan -skip-state-locking\n```\n\n## Best Practices\n\n1. **Always use plans before apply**\n   ```bash\n   terraform plan -out=tfplan\n   terraform apply tfplan\n   ```\n\n2. **Store state remotely**\n   - Enables team collaboration\n   - Automatic locking prevents corruption\n   - Backup through storage account replication\n\n3. **Use workspaces for environments**\n   ```bash\n   terraform workspace list\n   terraform workspace new prod\n   terraform workspace select prod\n   ```\n\n4. **Document all manual changes**\n   - Avoid making changes outside Terraform\n   - Always import back to state: `terraform import`\n\n5. **Review all plan outputs**\n   - Look for unexpected deletions\n   - Verify all variables are set correctly\n   - Check tags and naming conventions\n\n---\n\n**Need Help?**\n- Terraform Docs: https://registry.terraform.io/providers/hashicorp/azurerm/latest/docs\n- Azure CLI Docs:
